# 多线程

# 进程与线程

对Java工程师来说，Java编写的程序都运行在Java虚拟机(JVM)中，每当使用Java命令启动一个Java应用程序时，就会启动一个JVM进程。在这个JVM进程内部，所有Java程序代码都是以线程来运行的。理论上，在该进程的内部至少会启动两个线程，一个是main线程，另一个是GC（垃圾回收）线程。实际线程数量远远不止两个。JVM找到程序的入口点main()方法，然后运行main()方法，这样就产生了一个线程，这个线程被称为主线程。当main()方法结束后，主线程运行完成，JVM进程也随即退出。

# 线程有几种状态

Java 线程在运行的生命周期中的指定时刻只可能处于下面 6 种不同状态的其中一个状态：

- NEW: 初始状态，线程被创建出来但没有被调用 `start()` 。
- RUNNABLE: 运行状态，线程被调用了 `start()`等待运行的状态。
- BLOCKED：阻塞状态，需要等待锁释放。
- WAITING：等待状态，表示该线程需要等待其他线程做出一些特定动作（通知或中断）。
- TIME_WAITING：超时等待状态，可以在指定的时间后自行返回而不是像 WAITING 那样一直等待。
- TERMINATED：终止状态，表示该线程已经运行完毕。

# **多线程到底有几种实现方式 ?**

不同的角度来看可以有不同的表达

一般来说从使用的角度上有如下四种方式：

1. 继承Thread类，重写run()方法
2. 实现Runable接口（推荐因为只能单继承）
3. 实现Callable接口（需要返回值的时候使用）
4. 线程池

从实现原理来看，

创建线程只有一种方式：`new Thread()`

```java
// Thread 构造函数的几种重载
public Thread() {}
public Thread(Runnable target) {}
public Thread(ThreadGroup group, Runnable target) {}
```

 所有 `run()` 都来自 `Runnable` 接口

```java
public interface Runnable {
    void run();  // 所有 run() 的源头
}

public class Thread implements Runnable {
    private Runnable target;  // 可以持有另一个 Runnable

    @Override
    public void run() {
        if (target != null) {
            target.run();  // 调用传入的 Runnable
        }
    }
}
```

**Thread 的 run() 逻辑**：

- 子类重写了 `run()` → 执行子类的 `run()`
- 子类没重写 `run()` → 必须传入一个 `Runnable` → 执行 `Runnable.run()`

**FutureTask 的作用**：
FutureTask 将 Callable 与 Runnable 组合在一起，把 Runnable 的 `run()` 方法变成调用 Callable 的 `call()` 方法接收返回值，同时提供阻塞的 `get()` 方法获取结果。

```java
public class FutureTask<V> implements RunnableFuture<V> {
    private Callable<V> callable;  // 实际执行的任务
    private volatile int state;    // 任务状态
    private Object outcome;        // 结果或异常

    @Override
    public void run() {
        try {
            V result = callable.call();  // 执行 Callable
            set(result);                 // 保存结果
        } catch (Exception e) {
            setException(e);             // 保存异常
        }
    }
    public V get() throws InterruptedException, ExecutionException {
        int s = state;
        if (s <= COMPLETING)
            s = awaitDone(false, 0L);
        return report(s);//异常应该是在这里抛出
    }
}
```

实际项目中统一用线程池的方式管理多线程。也就是Runnable接口。

# **如何正确优雅的中断线程**

interrupt()其本身并不是一个强制打断线程的方法，其仅仅会修改线程的interrupt标志位，然后让线程周期性自行去读标志位，有秩序地终止任务，释放资源。

# 死锁

**产生死锁的四个必要条件：**

1. **互斥条件**：该资源任意一个时刻只由一个线程占用。
2. **请求与保持条件**：一个线程因请求资源而阻塞时，对已获得的资源保持不放。
3. **不剥夺条件**：线程已获得的资源在未使用完之前不能被其他线程强行剥夺，只有自己使用完毕后才释放资源。
4. **循环等待条件**：若干线程之间形成一种头尾相接的循环等待资源关系。

**如何检测死锁？**

- 使用`jmap`、`jstack`等命令查看 JVM 线程栈和堆内存的情况。如果有死锁，`jstack` 的输出中通常会有 `Found one Java-level deadlock:`的字样。

上面的是八股里的说明，但是事实项目上一般死锁应该是出现在数据库上，大家不会在jvm内部加同步，应该去数据库查看锁信息。
